# Compliance and Safety Guardrails

This document is not legal advice. It is a build-time risk checklist for keeping CryptoSeeds conservative, self-custodial, and clear.

## Product Language

Avoid:

- Guaranteed APY
- Guaranteed profit
- Risk-free yield
- Passive income
- Investment returns
- Profit share
- Token price appreciation
- AI money printer
- Copy-trade to earn
- Casino language

Prefer:

- Utility access
- Project participation
- Ecosystem rewards
- Claimable distribution
- Governance rights
- Impact updates
- Milestone reports
- Self-custodial execution
- Wallet-approved actions

## Project Participation

Project pages must show:

- Risk level
- Project status
- Documents
- Milestones
- Operator/owner
- Duration
- Participation terms
- Disclosures
- Smart contract address, when available

Donation projects must be clearly separated from reward-bearing or participation-based projects.

## SeedBot Terminal Rules

Never:

- Ask for seed phrases
- Store private keys
- Create backend-controlled user wallets
- Use unlimited delegated trading authority
- Hide transaction details
- Enable trading without clear user permission
- Promise profit
- Launch leverage or perps in MVP
- Build memecoin sniper mode in MVP

Required:

- Self-custodial wallet connection
- Plain-English transaction previews
- Token, route, slippage, fee, and risk display
- User wallet approval for execution
- Revocable guarded automation if added later
- Emergency disable/revoke control
- Activity logs

## Arcade / Experimental Zone

Do not implement token wagering, casino mechanics, games of chance, or prize systems without compliance review.

Safer alternatives:

- Cosmetic-only games
- Free-to-play mini-games
- Skill-based demos
- Paper rewards with no cash value
- Off-chain prototypes

## Architecture Safety

Protocol and application design should prioritize:

- Modular contracts
- Transparent fee calculations
- Emergency pause
- Clear authority controls
- Event logs
- Minimal admin trust
- No hidden minting
- No supply-changing mechanics
- Conservative reward logic
- Upgrade discipline
