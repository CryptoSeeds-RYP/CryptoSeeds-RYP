# Compliance and Safety Guardrails

This document is not legal advice. It is a build-time risk checklist for keeping CryptoSeeds conservative, self-custodial, decentralized where practical, and clear.

## Decentralization and Self-Custody

Self-custody and decentralization are mandatory product principles, not legal conclusions. They reduce custody, intermediary, and trust risk, but they do not guarantee that a feature is outside securities, commodities, money-transmission, financial-promotion, adviser, gambling, tax, or consumer-protection rules.

Build toward:

- User-controlled wallets
- User-directed transactions
- Wallet-approved signatures
- Open protocol accounts
- Transparent program state
- Objective routing parameters
- Revocable permissions
- Public fee logic
- Minimal admin control
- Multisig and timelock controls where admin authority remains
- On-chain events for staking, claims, voting, project participation, and permission changes

Do not build or market CryptoSeeds as a regulatory workaround. Public and in-app language must never suggest that self-custody lets the project ignore, bypass, evade, or opt out of applicable law.

Prefer:

- Designed to be self-custodial
- User-directed
- Non-custodial interface
- Protocol-based access
- Transparent transaction preparation
- Decentralization-minded architecture
- Reduced intermediary risk
- Legal review required before public launch

Avoid:

- Avoids financial services law
- Bypasses regulation
- Regulation-free
- Guaranteed compliant
- Outside securities law
- No legal risk
- Self-custody solves compliance

## Product Language

Avoid:

- Guaranteed APY
- Guaranteed profit
- Risk-free yield
- Passive income
- Investment returns
- Profit share
- Token price appreciation
- AI money printer
- Copy-trade to earn
- Casino language

Prefer:

- Utility access
- Project participation
- Ecosystem rewards
- Claimable distribution
- Governance rights
- Impact updates
- Milestone reports
- Self-custodial execution
- Wallet-approved actions

## Project Participation

Project pages must show:

- Risk level
- Project status
- Documents
- Milestones
- Operator/owner
- Duration
- Participation terms
- Disclosures
- Smart contract address, when available

Donation projects must be clearly separated from reward-bearing or participation-based projects.

## SeedBot Terminal Rules

Never:

- Ask for seed phrases
- Store private keys
- Create backend-controlled user wallets
- Use unlimited delegated trading authority
- Hide transaction details
- Enable trading without clear user permission
- Promise profit
- Provide discretionary account management
- Choose trades for a user without pre-disclosed objective parameters and explicit user approval
- Launch leverage or perps in MVP
- Build memecoin sniper mode in MVP

Required:

- Self-custodial wallet connection
- Plain-English transaction previews
- Token, route, slippage, fee, and risk display
- User wallet approval for execution
- Objective and pre-disclosed route parameters
- Revocable guarded automation if added later
- Emergency disable/revoke control
- Activity logs
- Legal review before any live success-fee, profit-fee, performance-fee, or guarded-automation model

## Arcade / Experimental Zone

Do not implement token wagering, casino mechanics, games of chance, or prize systems without compliance review.

Safer alternatives:

- Cosmetic-only games
- Free-to-play mini-games
- Skill-based demos
- Paper rewards with no cash value
- Off-chain prototypes

## Architecture Safety

Protocol and application design should prioritize:

- Modular contracts
- Transparent fee calculations
- Emergency pause
- Clear authority controls
- Event logs
- Minimal admin trust
- No hidden minting
- No supply-changing mechanics
- Conservative reward logic
- Upgrade discipline

## Legal-Review Triggers

These features must stay disabled, mocked, or document-only until reviewed:

- Custody of user funds
- Backend-held trading keys
- Pooled user assets controlled by the team
- Discretionary strategy execution
- Profit-based or performance-based fees
- Project revenue entitlements
- Tokenized SPV, equity, debt, or real-world asset rights
- Fiat on/off ramps
- Leverage, perps, or margin
- Copy trading
- Public strategy marketplaces
- Gambling, prize, or token-wager mechanics
- UK-facing or US-facing public financial promotions
